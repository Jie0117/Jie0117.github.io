設定log裡面顯示的使用者以及email

若你有傳遞 –global 參數，只需要做這工作一次，因為在此系統，不論 Git 做任何事都會採用此資訊。 若你想指定不同的名字或電子郵件給特定的專案，只需要在該專案資料夾內執行此命令，並確定未加上 –global 參數

windows系統會預設儲存於C:\user\.gitconfig

$ git config --global user.name "Jie Chen"
$ git config --global user.email Jiechen@example.com

指定文字編輯器

如果你想要在 Windows 系統上使用 Notepad++ 做為編輯器，你可以執行下列步驟：

$ git config --global core.editor "'C:/Program Files (x86)/Notepad++/notepad++.exe' -multiInst -nosession"

在現有資料夾中初始化git

指令執行後會在當前資料夾新增一個.git隱藏檔案

$ git init

複製現有repository

若你想要取得現有 Git repository的複本，那你需要使用的命令是 git clone

$ git clone [url] [本機指定路徑(選填)]

檢查檔案狀態

檔案的狀況只會有以下四種，它們的狀態可能是「Unmodified」, 「Modified」, 「Staged」, 「Untracked」，下圖為檔案狀態的生命週期

  • Unmodified（未修改）：這表示文件的內容與上一次提交時相同，沒有任何更改。這是最基本的狀態，意味著文件沒有被更改。

  • Modified（已修改）：這表示文件的內容已經被修改，但是這些修改還沒有被暫存到Git的暫存區（使用git add命令）。這些修改只存在於工作資料夾中。

  • Staged（已暫存）：這表示文件的修改已經被暫存到Git的暫存區，準備提交到版本庫（使用git commit命令）。在這個狀態下，文件的當前版本與上一次提交時的版本不同，但是已經暫存並準備好提交。

  • Untracked（未跟蹤）：這表示文件是新建的，並且尚未被Git跟蹤。這些文件存在於工作資料夾中，但是還沒有被添加到Git的版本控制中。通常情況下，這些文件需要使用git add命令將其添加到暫存區，然後再進行提交。 

$ git status
$ git status -s //簡潔的status輸出

上述指令會告知目前的檔案狀況

追蹤新的檔案

$ git add [檔案名稱]
$ git add . //會將所有Untracked和Modified的文件添加到暫存區
$ git add * //會將中所有Modified的文件添加到暫存區，但不包括Untracked的文件

檢視已暫存及未暫存的檔案

$ git diff // 可顯示檔案裡的哪些列被加入或刪除

提交你的暫存區

$ git commit //啟用文字編輯器提交訊息
$ git commit -m "Your commit message" //直接輸入提交訊息

第一列會啟動你所選定的文字編輯器 （由你的 Shell 的 $EDITOR 環境變數所指定——通常是 vim 或 emacs；你也可以使用 git config –global core.editor 命令指定任何一個你想使用文字編輯器）。

略過暫存區直接提交

$ git commit -a -m 'added new benchmarks'

在 git commit 命令加上 -a 選項，使 Git 在提交前自動預存所有已追蹤的檔案，讓你略過 git add 步驟

注意： 在提交之前，你不需要執行 git add 來暫存存檔案； 那是因為 -a 選項會納入所有已變更的檔案； 雖然很方便，但小心，有時候它會加入你並不想要的變更。

移除檔案

$ git rm [filename] //會從git版本控制跟資料夾刪除
$ git rm --cache [filename] //只刪除git版本控制，資料夾保留檔案

要從 Git 中刪除一個檔案，你需要將它從暫存區檔案中移除，然後再commit； git rm 命令可完成此工作，它同時也會將該檔案從工作資料夾中移除。

移動檔案or重新命名

$ git mv file_from file_to

有人可能對此命令感到困惑，命令中的mv表示"move"（移動），但在這個情況下，它被用來執行重命名操作，因為mv命令也可以用於將文件移動到不同的資料夾中。第一個參數README.md是你要重命名的文件的原始名稱，第二個參數README是重命名後的目標名稱。

其實它相當於執行下列命令：

$ mv README.md README
$ git rm README.md
$ git add README

檢視提交的歷史記錄

預設情況（未加任何選項）git log 以最新的commit會先被列出來，但是git log 命令有大量且多樣的資訊，在這裡我們會展示一些受歡迎的選項。能精確地找出你想搜尋的結果。

$ git log

最有用的選項之一是 -p，用來顯示每筆提交所做的修改內容； 你還可以加上 -2 選項，限制只輸出最後兩筆提交內容

$ git log -p -2

若想檢視每筆提交簡略的統計資訊，你可以使用 --stat 選項

$ git log --stat

另一個實用的選項是 --pretty， 用來改變原本預設輸出的格式，選項輸出的格式大致相同，但分別會少一些或者多一些資訊。例如：

  • oneline
  • short
  • full
  • fuller
$ git log --pretty=oneline

有趣的是format讓你可以指定自訂的輸出格式，當需要輸出給程式分析時特別有用。例如：

$ git log --pretty=format:"%h - %an, %ar : %s

下列是一些可選的格式

選項 輸出說明
%H 該提交 SHA-1 雜湊值
%h 該提交簡短的 SHA-1 雜湊值
%T 「樹（tree）」物件的 SHA-1 雜湊值
%t 「樹」物件簡短的 SHA-1 雜湊值
%P 親代（parent）提交的 SHA-1 雜湊值
%p 親代提交簡短的 SHA-1 雜湊值
%an 作者名字
%ae 作者電子郵件
%ad 作者日期（依據 –date 選項值而有不同的格式）
%ar 作者日期，相對時間格式
%cn 提交者名字
%ce 提交者電子郵件
%cd 提交者日期
%cr 提交者日期，相對時間格式。
%s 標題

復原

$ git commit -m 'initial commit'
$ git add forgotten_file
$ git commit --amend

這個命令會再次把暫存區拿來提交， 如果自從上次提交以來你沒有做過任何檔案修改，那麼整個快照看起來會與上次提交的一模一樣，唯一有可能更動的是提交訊息。

將已暫存的檔案移出暫存區

$ git reset HEAD [filename]

顯示你的遠端版本庫

$ git remote

新增遠端版本庫

$ git remote add [remote-name] [url]

從遠端獲取版本

$ git fetch [remote-name]

推送到遠端

$ git push [remote-name] [branch-name]

檢視遠端

$ git remote show [remote-name]

重新命名遠端

$ git remote rename [ori_name] [rename]

刪除遠端

$ git remote rm [remote-name]
